## AIPatterns

### 概述
- 该文总结了在大规模智能体系统中常见的17种“Agentic AI”设计模式，可按任务阶段与能力组合以构建稳健、可扩展、可观测的系统。
- 组合思路：先规划与调度，再并行探索与验证；以工具、检索与记忆增强事实性；以评审、投票与自检提升稳健性；以评估与遥测持续闭环优化。

### 设计模式详述（要点）
- 多智能体协作：
  - 适用：复杂多阶段/多专长任务。
  - 机制：基于角色/能力编排代理流水线或并行协同。
  - 要点：任务边界清晰、接口契约、失败隔离与重试。
- 集成决策/投票：
  - 适用：答案鲁棒性要求高的场景。
  - 机制：多候选采样，投票/加权/仲裁选优。
  - 要点：多样化生成、校准打分、成本‑质量平衡。
- 树状思维（ToT）：
  - 适用：需要分支探索与中间评估的推理问题。
  - 机制：并行扩展节点、评估回溯、剪枝选优。
  - 要点：启发式评估函数、宽/深度预算、缓存中间状态。
- ReAct 回路：
  - 适用：需要工具交互与环境反馈的任务。
  - 机制：思考→行动→再思考的循环策略。
  - 要点：动作空间约束、错误处理、观测归因。
- 反思/自纠（Reflexion）：
  - 适用：需要利用错误反馈持续改进的场景。
  - 机制：显式生成自评/纠错提示并重写输出。
  - 要点：错误信号收集、反思模板、迭代收敛控制。
- 计划‑执行‑验证（PEV）：
  - 适用：可被断言/测试/评审验证的任务链。
  - 机制：先计划，再执行，最后用可验证标准验收。
  - 要点：计划可检验化、失败回滚与重试策略。
- 元控制/调度（Meta‑Control）：
  - 适用：多代理/多工具/多策略的动态编排。
  - 机制：顶层控制器根据目标与度量进行策略选择。
  - 要点：路由策略、带宽/成本预算、在线学习。
- 任务分解与规划：
  - 适用：大任务需并行/复用的拆分场景。
  - 机制：明确先后关系、依赖与验收条件。
  - 要点：最小可交付子任务、数据与接口对齐。
- 工具使用（Tool‑Use）：
  - 适用：需要事实性、计算或行动能力增强。
  - 机制：受控工具调用（API/DB/代码执行等）。
  - 要点：工具契约、安全权限、速率与超时控制。
- 检索增强与记忆（RAG/Memory）：
  - 适用：长上下文与跨轮一致性需求。
  - 机制：向量检索+短/长时记忆融合。
  - 要点：索引质量、去重与时效、缓存/刷新策略。
- 角色/职责链（Roles & Chains）：
  - 适用：需要可审计与可控质量的产线式流程。
  - 机制：撰稿‑评审‑修订等角色链路。
  - 要点：明确交付工件、评审标准与回路。
- 代理仿真/社会模拟：
  - 适用：策略评估、数据生成、鲁棒性测试。
  - 机制：多代理在环境中交互涌现行为。
  - 要点：环境逼真度、指标设计、干扰注入。
- 探索‑利用权衡：
  - 适用：提示/工具/策略空间的持续寻优。
  - 机制：Bandit/贝叶斯优化等自适应探索。
  - 要点：冷启动、后验不确定性、停机准则。
- 安全护栏与策略：
  - 适用：合规/品牌/安全敏感业务。
  - 机制：内容策略、权限与约束解码器。
  - 要点：策略更新灰度、误拦与漏拦监控。
- 自一致性与自检：
  - 适用：减少幻觉与推理偶然性的场景。
  - 机制：多思路采样与一致性选择/规则自检。
  - 要点：采样温度、覆盖度、仲裁标准。
- 反事实/对抗评估：
  - 适用：发现脆弱点并驱动加固。
  - 机制：对抗样例与反事实查询。
  - 要点：覆盖关键边界、回归保护、红队基线。
- 在线评估与遥测：
  - 适用：持续监控与闭环优化。
  - 机制：任务得分、用户交互与A/B测试。
  - 要点：延迟/成本/质量三角、失效告警、数据回流。
